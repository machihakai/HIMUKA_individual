# 株式会社ヒムカ 会社サイト 開発計画

## 1. ゴールと背景
- `HIMUKA_info.md` に記載された会社概要・経営理念・取扱メーカー・販売先・採用情報などを整理し、1ページ完結のシンプルなコーポレートサイトを構築する。
- PC/タブレット/スマートフォンの主要幅で閲覧しやすいレイアウトとし、問い合わせ情報へ誘導しやすくする。

## 2. 技術選定
| 項目 | 選定 | 理由 |
| --- | --- | --- |
| マークアップ | 素の HTML5 | 規模が小さいためフレームワーク不要。読み込み高速、ホスティングも容易。 |
| スタイル | 素の CSS（`styles.css`） | デザイン要件がシンプルなため。カスタムプロパティと Flex/Grid を活用しレスポンシブ最適化。 |
| JavaScript | バニラ JS（必要最小限） | スクロール連動のナビゲーションや軽微な UI 効果のみ。 |
| アセット管理 | `assets/` 配下に `HIMUKA_logo.png` をコピー | ロゴの再利用を容易にするため。 |
| ビルド/開発サーバー | なし（ファイル直開 or 任意の静的サーバー） | 依存を増やさず、Windows 上でも即確認可能。 |

## 3. 情報設計（ページ構成）
1. **ヒーローセクション**：ロゴ、キャッチコピー、主要 CTA（問い合わせ誘導）。
2. **会社概要**：商号、代表者、所在地、設立、資本金、取引銀行、許可番号、建設業種別。
3. **経営理念/企業方針**：本文と 6 項目の箇条書き。
4. **主要取扱メーカー**：プラント資材／土木資材をカラム分けリスト化。
5. **主要販売先**：二列レイアウトで主要企業を表示。
6. **取扱商品**：アイコン風のカードで9カテゴリを紹介。
7. **協力業者**：業務内容とセットでリスト化。
8. **アクセス**：住所、電話、FAX をまとめ、Google Maps へのリンクボタン。
9. **お問い合わせ**：メールアドレスをスパム対策済みの形で表示し、電話・FAX と合わせた CTA を設置。
10. **採用情報**：募集概要、条件、メッセージ、応募誘導ボタン。
11. **フッター**：著作表示と再度の連絡先。

## 4. デザイン方針
- ブランドカラー：ロゴに合わせ濃紺＋アクセントで深緑を想定。背景はホワイト/ライトグレーで可読性を重視。
- レイアウト：モバイルファーストで組み立て、`min(1200px, 90vw)` のコンテナ幅とし、1024px 以上で 2〜3 カラム、768px 未満で 1 カラム。
- タイポグラフィ：日本語は游ゴシック/ヒラギノ/游明朝系のフォールバック、英数は `Segoe UI`, `Roboto` など。
- 視覚要素：セクションの区切りに淡いグラデーションや斜めボーダーを使い、重厚感と信頼性を演出。
- 余白とタップ領域：モバイルで 16px 以上、PC で 24px 以上のパディングを確保し、ボタン高さは 44px 以上に統一。

## 5. 実装ステップ
1. **プロジェクト雛形の作成**
   - ルートに `index.html`・`styles.css`・`script.js` を作成。
   - `assets/HIMUKA_logo.png` を配置。
2. **基本マークアップ**
   - 各セクションの HTML 骨組みを作り、`HIMUKA_info.md` からテキストを転記・整形。
3. **共通レイアウトとテーマカラー**
   - CSS でベースタイポ・配色・コンテナ幅・レスポンシブブレークポイントを定義。
4. **各セクションのスタイリング**
   - グリッドやカード表現を用いて一覧・表を整理。
   - ヒーローやアクセスなど視覚的に重要な箇所を優先して装飾。
5. **軽微なインタラクション**
   - ナビゲーションのスムーススクロール、CTA ボタンのホバー効果などを `script.js` で実装。
6. **最終調整と検証**
   - PC/タブレット/モバイルでレイアウト確認。
   - テキストの誤植、アクセシビリティ（コントラスト比、alt 属性）をチェック。

## 6. テスト・公開の目安
- ブラウザ検証：最新の Chrome / Edge / Firefox で表示確認。
- レスポンシブ：375px, 768px, 1024px, 1440px 幅で簡易チェック。
- 公開方法：静的ホスティング（GitHub Pages, Netlify, Vercel など）または既存サーバーにファイルを配置。

## 7. セキュリティ・配信対策
1. **メールアドレスの難読化**：`info[at]example.co.jp` など記号置換、または JavaScript で DOM 挿入し、ソース直書きを避ける。
2. **HTTPS 配信**：CDN / 静的ホスティングは必ず HTTPS 対応のサービスを選び、画像・スクリプトを同一オリジンもしくは HTTPS 経由で配信。
3. **適切なヘッダー**：`Content-Type`, `Cache-Control` などが自動設定されるホスティングを選択し、必要なら `headers` 設定ファイルで明示。
4. **外部サービス連携時の確認**：地図やフォーム埋め込みを行う場合は、提供元の CSP/利用規約を確認し、埋め込みコードを最新に保つ。

## 8. SEO 基本対策
1. **メタ情報**：`<title>` に「株式会社ヒムカ | 公式サイト」、`<meta name="description">` に耐食機器材料の加工・販売などの事業概要を記載。
2. **見出し構造**：`<h1>` は会社名＋キャッチコピー、各セクションを順序立てた `<h2>`/`<h3>` でラベル付けし、クローラーに内容を伝える。
3. **テキスト化**：会社概要・所在地・事業内容・連絡先は画像化せずテキストで配置し、検索インデックスに確実に含める。
4. **OGP/言語属性**：`<html lang="ja">`、主要 SNS 向けに `og:title`, `og:description`, `og:image` を設定してシェア時の表示を最適化。

## 9. 追加で検討したいコンテンツ
1. **お知らせ／ニュース**：許可更新、施工実績、採用イベントなどの最新情報を一覧化し、会社の活動量を伝える。
2. **施工実績・導入事例**：写真＋課題／解決策の簡潔な紹介で信頼度を高める。
3. **品質・安全・環境への取り組み**：ISO・SDGs・BCP などの認証・方針があればまとめる。
4. **ダウンロード資料**：会社案内 PDF や製品カタログを設置して営業活動を支援。
5. **FAQ**：問い合わせの多い質問を事前に提示し、営業・総務の負担を軽減。
